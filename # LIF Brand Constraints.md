# LIF Brand Constraints  
  
## Project  
**Lesson Intelligence Framework (LIF) Landing Page**  
  
## Primary Goal  
Convert educators into subscribers who receive the **LIF Teacher’s Guide**.  
  
## Audience  
- Teachers (primary)  
- Instructional leaders  
- Conference attendees  
- Educators curious about **AI-resilient, thinking-driven lesson design**  
  
---  
  
## Brand Personality  
- Serious  
- Professional  
- Calm  
- Evidence-grounded  
- Teacher-respectful  
  
## Voice  
- Warm, educator-focused, forward-thinking  
- Practical and specific (classroom-facing)  
- No hype, no guru tone, no marketing jargon  
- Write like a colleague sharing a reliable framework  
  
---  
  
## Brand Colors  
  
### Core Palette  
- **Primary (Navy):** `#0B132B`  
- **Accent (Purple):** `#5B2EFF`  
- **Background (Light Neutral):** `#F7F8FA` (optional gradient end: `#E9EEF6`)  
- **Text (Dark Charcoal):** `#111827` (use `#0B132B` for headings if desired)  
  
### Support Colors (UI)  
- **Secondary text:** `#6B7280`  
- **Dividers / borders:** `#E6E8EF`  
- **Card background:** `#FFFFFF`  
  
### Color Usage Rules  
- Default to **light mode** for the landing page.  
- Use **purple** for CTAs, links, and “signal” elements (underlines, small icons, badges).  
- Use **navy** for headings and key anchors.  
- Avoid heavy color blocks; use whitespace + subtle borders for structure.  
- Maintain accessible contrast (text must be readable on all backgrounds).  
  
---  
  
## Typography  
  
### Type System  
- **Heading font (Display):** **Anton**  
- **Body font (Sans-serif):** **Inter**  
  
### Usage Rules  
- Anton: only for **H1/H2** and short section headers (1 line when possible).  
- Inter: all body text, subheads, captions, UI labels.  
- Keep line lengths readable (mobile-first), avoid dense paragraphs.  
  
---  
  
## Visual Rules  
  
### Layout + Structure  
- Minimal, clean, spacious  
- Strong hierarchy: one primary message per section  
- Clear rhythm: section spacing, consistent padding, consistent card styles  
- Use **cards** for: benefits, framework layers, testimonials/quotes, FAQs, opt-in blocks  
- Mobile-first: single-column layout by default; upgrade to two-column only where it helps scanning  
  
### Styling  
- No clutter  
- No generic stock-hero look (avoid cliché classroom imagery)  
- Prefer simple geometric accents, clean icons, subtle separators  
- Subtle motion only (micro-interactions, soft fades, small hover states)  
  
### Accessibility  
- High contrast for text  
- Clear focus states on interactive elements  
- Adequate font sizes and tap targets  
- Avoid low-contrast gray-on-gray UI  
  
---  
  
## Page Intent (What the landing page must do)  
- Immediately communicate:  
  1) What LIF is    
  2) Who it’s for    
  3) The outcome: lessons where thinking is **required, visible, guided**    
  4) The CTA: subscribe to receive the Teacher’s Guide  
- Remove friction: keep the opt-in simple, visible, and repeated at natural decision points.
